Angular Exercises

Simple Component (1/3)

  1. Create a new module Books (ng g module Books).
  2. Go to the books directory and create a new component BookDetails (ng g component BookDetails).
  3. Export the BookDetailsComponent from the BooksModule - both of them you've just created.
  4. Import the BooksModule in the AppModule. Remove the AppComponent.title property.
  5. Add <app-book-details> to the AppComponent's template. book-details works! appears on the page.

Simple Component (2/3)

  1. Create a class Book in the books directory (ng g class Book). Add the id, author and title properties.
  2. Add a public BookDetailsComponent.currentBook property of type Book and initialize the property with an empty Book object in the constructor.
  3. Take part in the OnInit lifecycle hook (an empty ngOnInit() method should have been generated) setting the author and title properties of currentBook to any values of your choice.

Simple Component (3/3)

  1. Create a Book form (in the file book-details.component.html).
  2. For now don't use <input>s. Just display labels and values of the authors and title properties (using the interpolation {{}}).
  3. Add some component's styling (to the file book-details.component.scss) , e.g. make the labels' font weight bold. Make use of the :host pseudo selector.

Master/Details Pattern (1/6)

  1. Let's assume we're in the app/books directory.
  2. Create a new component BookOverview (ng g component BookOverview) and export it from the BooksModule.
  3. Add <app-book-overview> to the AppComponent's template. book-overview works! should appear on the page.

Master/Details Pattern (2/6)

  1. Add public properties to the BookOverviewComponent class:
    • the array books which keeps available books,
    • the selectedBook which keeps currently selected book from available ones.
  2. Initialize the books array in the constructor with an empty array.
  3. Add some book entries to the books array (In the OnInit lifecycle hook). For book creation add a static convenience method to the Book class, e.g.: Book.from(authors: string, title: string): Book

Master/Details Pattern (3/6)

    Extend the BookOverviewComponent class:
  1. Implement the isBookSelected(book: Book): boolean method which checks if the book passed is the selected one.
  2. Implement the selectBook(book: Book): void method which sets the passed book as the selected one.

Master/Details Pattern (4/6)

  1. Add two <div> elements to the overview component.
  2. To the first <div> add a table with rows. Stamp out the table's rows iterating over the BookOverviewComponent.books. Make use of the *ngFor directive. For each book element show its author and title as well as the row number (in the first column).

Master/Details Pattern (5/6)

    1. Add the following bindings to the table rows:
    • the (click) event binding to the selectBook(book:Book):void method,
    • the class binding [class.table-active] to the isBookSelected(book: Book): boolean method.
  2. Put the <app-book-details> element in the second <div> and add the [book] property binding to the selectedBook.

Master/Details Pattern (6/6)

  1. Go to the BookDetailsComponent class and remove all its content except for the currentBook property. Rename it to book.
  2. Add the @Input() decorator to the book property.
  3. Go to the BookDetailsComponent's template. The <div> element containing the BookDetailsComponent should only be displayed if the book property is set. Use the *ngIf directive.

Routing (1/6)

    Create the app/app-routing.module.ts file. Export the appRoutes constant which is going to keep the application routes so that:
  • all paths starts with book-app,
  • the books path (relative to book-app) leads to the BookOverviewComponent,
  • the book path leads to the BookDetailsComponent (when adding a new book),
  • the book/:id path leads to the BookDetailsComponent (when editing an existing book),
  • the empty path ('') redirects to /book-app/books.

We prepared a server for you.
User the following commands for startup

  • "npm run start" - starts angular server with reverse proxy
  • "npm run server" - starts the rest server

Possible REST calls:

  • get http://localhost:4200/services/rest/books
  • get http://localhost:4200/services/rest/book/:id
  • post http://localhost:4200/services/rest/book
