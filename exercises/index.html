Simple Component

  1. Create a new module
  2. Create a new component
  3. Import the component into the module
  4. Show the component
  5. Show dummy data in the component

Fallback branch: exercise/1-project-seed
Solution branch: exercise/2-simple-component

Git Cheat Sheet

Simple Component (1/3)

  1. Create a new module Movies (ng g m movies).
  2. Create MovieDetails component in movies directory (ng g c movies/movie-details).
  3. Export the MovieDetailsComponent from the MoviesModule - both of them you've just created.
  4. Import the MoviesModule in the AppModule. Remove the AppComponent.title property.
  5. Add <app-movie-details> to the AppComponent's template. movie-details works! appears on the page.

Simple Component (2/3)

  1. Create an interface Movie in the movies directory (ng g interface movies/movie). Add the id, directors , title, description and year properties.
  2. Add a public currentMovie property of type Movie to the MovieDetailsComponent
  3. In the OnInit lifecycle hook (an empty ngOnInit() method should have been generated) assign a newly created Movie object with the director, title and other properties with any values of your choice. {id:1, title: 'a movie',...}

Simple Component (3/3)

  1. in the file movie-details.component.html
    • Create a <div>
    • Display the movies properties inside the <div>
    • For now don't use <input>. Just display labels and values of the director and title properties (using the interpolation {{currentMovie.title}}).
  2. Add some component's styling (to the file movie-details.component.scss) , e.g. make the labels' font weight bold. Make use of the :host pseudo selector.

Git Cheat Sheet

Service (1/5)

  1. Create a new Movie service (ng g service movies/movie) with the private movies property.
  2. Initialize the property with movies from MovieOverviewComponent.ngOnInit() the service.

Service (2/5)

  1. Implement findAll(): Observable<Movie[]> method which returns the movie array as an Observable.
  2. Implement save(movieToSave: Movie): Observable<Movie>. Move in the logic for updating/saving movies and return the created/updated movie.

Service (3/5)

  1. Make use of dependency injection and inject MovieService in MovieOverviewComponent's constructor.
  2. In MovieOverviewComponent.ngOnInit() call the MovieService.findAll() method. Subscribe to the returned Observable to initialize the movie list.
  3. Use the save(movieToSave: Movie): Observable<Movie> in the MovieOverviewComponent.onMovieUpdated and reload the movies after the update Observable completes.

Service (4/5)

  1. In the MovieOverviewComponent change the movies array to movies$ of type Observable<Movie[]>. Do not subscribe to the MovieService.findAll() but assign the resulting Observable to the movies$ property.
  2. In the MovieOverviewComponent template use the async pipe in the *ngFor to automatically subscribe to the Observable.

Service (5/5)

    Extend the MovieService:
  1. Implement the findOne(id: number): Observable<Movie> method which finds a book by the ID passed and returns its copy.

Service (extra)

Forms

This exercise is divided into two parts:

  • A: Reactive Forms
  • B: Template-driven Forms

Fallback branch: exercise/forms
Solution branch: exercise/forms-solution

Git Cheat Sheet

Forms (reactive Forms 1/4)

  • Import the ReactiveFormsModule in your AppModule.
  • Create a Form-Model with following structure by using FormGroup and FormControl: 
{ 
name: { 
  firstname: string, 
  lastname: string }, 
email: string,
password: string, 
passwordRepeat: string 
}

Forms (reactive Forms 2/4)

  • EMail should contain the default value "example@mail.com"
  • When the user submits the form, trigger the prepared function in ReactiveFormComponent and pass the form content.

Forms (reactive Forms 3/4)

  • add following validators:
    • name is validated for a minimum length of 4 chars
    • name contains only a-z,A-Z," "
    • name and password are required fields
    • password should have at least 8 chars
    • repeat-password-field is validated for exact match with the password-field. (definde validator on your own)

Forms (reactive Forms 4/4)

  • Show the prepared textual descriptions (currently hidden with display:none) on validation errors for each field validated to give the user guidance on how to fix the errror. Use *ngIf to hide messages and use null conditional notation to check for different validation-errors.

Forms (template-driven Forms 1/4)

  • Import the FormsModule in your AppModule.
  • use the ngModel directive to create a user-object in the form 
    {
"firstname": "",
"lastname": "",
"eMail": "yourName@domain.com",
"password": "",
"passwordRepeat": ""
}

Forms (template-driven Forms 2/4)

  • when the user clicks the submit-button, call the prepared function "onSubmitTemplateBased" and pass the whole form content as parameter.
  • Use one-way binding to fill the eMail input field with exampleEmail-property of the according component

Forms (template-driven Forms 3/4)

  • Add Form directives to the html-template of the TemplateFormComponent, to achieve following:
    • name is validated for a minimum length of 4 chars
    • name contains only a-z,A-Z," ", use a pattern-validator
    • name and password are required fields
    • password should have at least 8 characters
    • "Repeat Password" should use ValidateEqualDirective to validate it matches the password-field

Forms (template-driven Forms 4/4)

  • Show the prepared error-messages only when according validation-error occur. check the state of an input-field by using a local template variable. Show the whole alert only if an validation-error for the according field exists.
  • Figure out, why input fields are marked as valid/invalid by the red/blue indicaror-bar?
